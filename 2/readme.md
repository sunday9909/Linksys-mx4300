这份脚本是专为 Linksys MX4300 (LN1301) 路由器在 OpenWrt 固件环境下设计的。它利用路由器闲置的 MTD30 (app2_data) 分区（约 353MB），通过 UBI 卷技术将其挂载为系统的全局 /overlay 层。
以下是为你准备的脚本简介与使用说明：
------------------------------
📝 脚本简介：Linksys MX4300 存储空间全量扩容方案
脚本名称： expand_overlay_mtd30.sh
适用硬件： Linksys MX4300 / LN1301
适用系统： 已刷入 OpenWrt 的固件
核心原理：

   1. 激活分区：将原厂预留的 mtd30 分区从隐藏状态激活并初始化为 UBIFS 格式。
   2. 无损迁移：自动将当前 /overlay 内的所有配置、插件、拨号信息同步到新分区，确保扩容后配置不丢失。
   3. 全局覆盖：将 MTD30 挂载为系统的全局存储层。扩容后，无论是安装 OpenClash 核心、Docker 镜像还是系统补丁，都将直接使用这 350MB 的大容量空间。

------------------------------
🛠 使用说明1. 执行前准备（必读）

* 网络连接：请确保路由器已联网，脚本会自动检查并尝试安装必要的工具包（ubi-utils）。
* 固件兼容：请确认你的固件分区表中存在 mtd30。你可以通过命令 cat /proc/mtd | grep mtd30 验证。
* 备份建议：虽然脚本带数据迁移功能，但重大底层操作建议先在 OpenWrt 后台备份一份 generate_config。

2. 快速部署步骤
在 SSH 终端（如 PuTTY 或 Termius）中依次输入以下操作：

   1. 创建并运行脚本：
   直接粘贴之前提供给你的脚本内容（或运行下方的简化版一行命令）：
   
   # 假设你已将脚本保存并上传至 /tmp/expand.sh
   chmod +x /tmp/expand.sh
   /tmp/expand.sh
   
   2. 确认交互：
   脚本会提示：确认执行并重启路由器？(y/n)。输入 y 并回车。
   3. 等待重启：
   脚本会自动完成：格式化 -> 迁移数据 -> 修改启动项 -> 自动重启。

3. 验证扩容结果
重启完成后，重新连接 SSH，输入以下命令查看：

* 查看空间大小：

df -h /overlay

预期结果：Size 字段显示约为 300MB - 350MB 左右。
* 查看挂载状态：

mount | grep ubi1_0

预期结果：显示 ubi1_0 on /overlay type ubifs ...

------------------------------
⚠️ 注意事项与常见问题

* 为什么不直接改 OpenClash 路径？
挂载到全局 /overlay 比修改单个插件路径更稳定。它解决了“根目录空间不足”的根本问题，OpenClash 会自动识别到大空间，无需任何额外配置。
* 如果我想恢复原样怎么办？
只需编辑 /etc/rc.local，删除关于 mtd30 和 ubiattach 的相关行，然后重启即可。系统会自动回滚到原来的小容量 Flash 存储。
